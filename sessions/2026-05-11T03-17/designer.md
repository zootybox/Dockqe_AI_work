# 🎨 Designer — 키워드 분석하기

🎨 Designer: 작업 시작합니다.

요청하신 '애드센스 수익을 위한 블로그 포스팅용 키워드 수집기' 개발을 위한 첫 단계로, 현재 회사 목표와 기존 연구 결과를 바탕으로 키워드 분석의 방향성을 설정하고, 향후 디자인 시스템 구축에 필요한 구조를 제안합니다.

현재는 실제 키워드 데이터가 없으므로, **키워드 수집기/분석 시스템이 갖춰야 할 핵심 구조와 디자인 프레임워크**를 먼저 설계하겠습니다.

### 1. 키워드 분석 목표 설정 (Designer 관점)

키워드 수집기의 궁극적인 목표는 단순히 키워드를 모으는 것이 아니라, **'수익 창출에 가장 효과적인 콘텐츠 주제'를 시각적으로 즉시 파악**할 수 있도록 하는 것입니다.

*   **핵심 목표:** 정보성 키워드 (70%) vs. 문제 해결형 키워드 (30%)의 비율을 시각화하여, 콘텐츠 전략의 우선순위를 명확히 제시한다.
*   **사용자 경험(UX) 목표:** 사용자가 복잡한 데이터 분석 없이, 어떤 키워드를 선택해야 할지 직관적으로 판단하게 한다.

### 2. 키워드 데이터 구조 제안 (비주얼 시스템 기초)

수집된 키워드를 분류하고 관리하기 위한 데이터 필드와 시각적 구분을 제안합니다.

| 필드명 | 설명 | 디자인 시 고려사항 |
| :--- | :--- | :--- |
| **Keyword** | 수집된 핵심 키워드 | 가장 큰 폰트, 강조 처리 |
| **Type** | 키워드 유형 (정보성 / 문제 해결형) | 색상 코드로 명확히 구분 (예: 정보성=파랑, 문제 해결형=주황) |
| **Intent Score** | 잠재적 수익/검색량 점수 (추가 분석 필요) | 막대 그래프 또는 별점 시스템으로 시각화 |
| **Content Angle** | 콘텐츠가 다룰 핵심 주제 (예: A 프로그램 사용법, B 문제 해결 방법) | 태그(Tag) 또는 아이콘으로 분류 |
| **Priority** | 콘텐츠 우선순위 (High/Medium/Low) | 색상 농도로 표현 (진한 색 = 우선순위 높음) |

### 3. 썸네일 컨셉 시뮬레이션 (키워드 기반)

수집된 키워드를 바탕으로 콘텐츠 썸네일의 기본 컨셉 3안을 제안합니다. (실제 키워드가 주어지지 않았으므로, 일반적인 수익화 콘텐츠의 디자인 패턴을 적용합니다.)

**컨셉 1: 데이터 기반 접근 (정보성 강조)**
*   **주요 요소:** 키워드 관련 통계/차트 이미지 + 명확한 텍스트 제목
*   **색상 팔레트:** 차분한 블루(신뢰)와 화이트(명료함). 강조색으로 밝은 녹색 사용.
*   **레이아웃:** 좌측에 통계 시각화, 우측에 핵심 키워드 및 제목 배치.

**컨셉 2: 문제 해결 강조 (Action 지향)**
*   **주요 요소:** 'Before & After' 또는 '문제점 제시 → 해결책 제시' 흐름 강조.
*   **색상 팔레트:** 대비가 강한 오렌지/레드(긴급성/문제)와 딥 그레이(전문성).
*   **레이아웃:** 화면을 세로로 분할하여 상단에 문제를, 하단에 해결책을 배치.

**컨셉 3: 시스템/툴 강조 (프로그램 중심)**
*   **주요 요소:** 프로그램/시스템의 깔끔한 UI 스크린샷 또는 아이콘 강조.
*   **색상 팔레트:** 모던한 그레이/블랙 기반에 브랜드 컬러(추후 확정)를 포인트로 사용.
*   **레이아웃:** 중앙에 핵심 툴 이미지, 주변에 키워드 태그를 깔끔하게 배치.
