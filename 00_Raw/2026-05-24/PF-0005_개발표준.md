# 📝 PublishFlow 개발 및 SEO 표준

## 📌 한 줄 통찰 (Agent Directive)
> 검색 엔진에 최적화된 마크업과 신뢰도 높은 JSON-LD 메타데이터 표준을 모든 콘텐츠와 웹 도구에 적용하여 검색 유입을 극대화한다.

## 🚀 SEO (검색 최적화) 가이드라인

### 1. 글 제목 및 메타 타이틀 규격
- **길이:** 공백 포함 55자 ~ 60자 권장
- **구조:** 핵심 키워드가 반드시 맨 앞에 오도록 배치하며, 구분자 `:`를 활용함 (예: `정부지원금 안내: 청년 월세 특별지원 대상 및 신청방법`)

### 2. 구조화 데이터 (JSON-LD) 삽입 필수
- 모든 웹 도구 페이지에는 `WebApplication` 또는 `FAQPage` 스키마를 JSON-LD 형식으로 헤더에 동적으로 생성하여 삽입합니다.

## 💻 코딩 표준 및 컨벤션

### 1. JavaScript 인젝션 보안 방침
- modutools의 광고 로직 및 API 키 보호 코드 작성 시, 코드가 평문으로 노출되어 어뷰징당하지 않도록 유의합니다.
- `eval(atob("Base64String"))` 패턴을 사용할 때, 한글은 반드시 이스케이프 유니코드로 변환 후 UTF-8 바이트 어레이 기준 인코딩을 적용합니다.

### 2. 파일 변경 규칙
- 에이전트가 코드를 편집할 때는 반드시 변경 전에 대상 파일을 읽어(`read_file` 또는 이에 대응하는 액션) 원본의 상태를 완벽히 숙지한 후 안전하게 치환(Fuzzy 매칭 활용)을 수행합니다.
- 작업 완료 후, 변경 사항은 자동으로 git sync 파이프라인을 통과하여 성재님의 저장소에 커밋/푸시되도록 설계합니다.
